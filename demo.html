Introducción

Comience a usar FireWorks, el framework de desarrollo frontend modular ideal para crear sitios web adaptativos a dispositivos móviles sin esfuerzo y ajustándose a lo que necesite. Al ser modular tiene opción de elegir qué elementos quieres tener y reducir el peso y tiempos de carga de tu web.

Inicio rápido

¿Desea agregar FireWorks rápidamente a su proyecto? Utilice nuestro CDN.

¿Necesita descargar los archivos fuente o desea elegir los componentes que necesita? Diríjase a la página de descargas.

CSS

Copie y pegue la hoja de estilo <link> en su <head> antes que el resto de hojas de estilo para cargar nuestro CSS. Ejemplo:

<link rel= "stylesheet" href= "style/fireworks.min.css">
JS

Algunos de nuestros componentes requieren el uso de JavaScript para funcionar. Además hay utilidades que hacen del proceso de desarrollo de una aplicación una tarea mas cómoda. A diferencia de otros frameworks, FireWorks no requiere de librerías externas, mejorando así la facilidad de implementación y la carga de ejecución. Para importarlo sólo tiene que bajarlo desde aquí y guardarlo en su proyecto. Ejemplo:

<script src="js/fireworks.js">
Configuraciones globales importantes

FireWorks emplea una serie de estilos y configuraciones globales importantes que deberá tener en cuenta al usarlo, los cuales están orientados mayormente a la normalización de estilos entre navegadores pero no exclusivamente.

FireWorks recomienda el uso del estándar HTML5 empezando por el doctype HTML5. Es, hasta la fecha, el doctype más cómodo de implementar y el primer paso para el uso del mencionado estándar:

<!doctype html> <html lang= "es"> ... </html>
Maquetación responsiva

FireWorks se desarrolla bajo la metodología mobile-first, una estrategia en la que se optimiza el código para dispositivos móviles primero y luego se escalan componentes según sea necesario usando mediaqueries en CSS. Para garantizar una reproducción adecuada y un zoom táctil para todos los dispositivos, agregue la metaetiqueta de la ventana sensible a su <head>:

<meta name= "viewport" content= "width=device-width, initial-scale=1, shrink-to-fit=no">

Puede deshabilitar las funciones de ampliación en los dispositivos móviles agregando user-scalable=no a la metaetiqueta. Esto desactiva el zoom, lo que significa que los usuarios solo pueden desplazarse, y hace que su sitio se sienta un poco más como una aplicación nativa aunque no siempre es recomendable.

<meta name= "viewport" content= "width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no">
Tamaño de caja

Para una mejor adaptación del CSS concerniente al tamaño de los elementos y su distribución se ha optado por el valor global de box-sizing a border-box en lugar de content-box. Puede causar confictos con software de terceros como Google Maps y Google Custom Search Engine por tanto, si fuese necesario, se pueden cambiar los componentes que se deseen usando la clase .content-box o creando una clase en su CSS:

.some-element { box-sizing : content-box ; }

Obtenga más información sobre el modelo de caja y el tamaño en CSS Tricks.

Elementos en línea

Para resaltar un segmento de texto a su relevancia en otro contexto, use la etiqueta <mark> o ahorre el valor semántico usando la clase .mark. Ejemplo:

Para tratar texto como eliminado se usará la etiqueta <del>.Para indicar segmentos que ya no son relevantes, use la etiqueta <s>. Por el contrario si lo que desea es únicamente tachar algo, use la clase .strike. Ejemplo:

Para tratar texto como modificado o se trata de una nueva inserción se usará la etiqueta <ins>. En caso de que se quiera subrayar algo, use la clase .underline. Ejemplo:

Se pueden usar con las clases .fg-recommend para aplicarlo como color de fuente y .bg-recommend como color de fondo. Lo mismo con .fg-info, .bg-info, .fg-success, .bg-success, .fg-alert, .bg-alert, .fg-warning, .bg-warning, .fg-error y .bg-error. Si se quiere aplicar como color de borde se pueden agregar a las palabras 'recommend','info','success','alert','warning' y 'error' también se les pueden aplicar los prefijos: 'border-' para todos los bordes, 'border-x' para los bordes izquierdo y derecho, 'border-y' para los bordes superior e inferior o 'border-top','border-left','border-bottom' y 'border-right' para tratar los bordes individuales.
